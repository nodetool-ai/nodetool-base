## 2025-04-25 - C-Level List Aggregation Optimizations
**Learning:** In Python, replacing `all(isinstance(x, (int, float)))` O(N) explicit type checks with `try...except TypeError` surrounding C-optimized built-ins like `sum()`, `min()`, and `max()` offers ~10-15x performance improvements for large lists. However, when optimizing `Product` calculations that use `reduce(lambda x, y: x * y, values)` or `math.prod()`, it is DANGEROUS to remove the upfront O(N) type check. Without it, standard string multiplication in Python (e.g., `10**9 * 'a'`) can easily be triggered, leading to a MemoryError / DoS. Upfront checking is strictly required for `Product`.
**Action:** Always verify `Product` or multiplication operations manually. Do not apply blanket EAFP patterns to them without ensuring sequence repetition cannot cause memory exhaustion.
